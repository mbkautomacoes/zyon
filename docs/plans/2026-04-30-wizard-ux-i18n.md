# Plan — Setup Wizard UX & Provider-Agnostic Copy

**Date:** 2026-04-30
**Owner:** trilha-1-multimodal
**Status:** draft

## Goal

Make `setup_config.py` (the interactive wizard invoked by `bootstrap.py` when `config.py` is missing) friendlier, vendor-neutral, and English-default.

## Why

1. **Current copy is Portuguese-only.** OSS distribution (post v1.1.0) means non-PT users land on the wizard and see `Configuracao Interativa`, `Preencha os dados abaixo`, etc. English is the standard default for OSS tooling.
2. **Each prompt is bare.** The user sees `Nome da instancia` with no idea what an instance is, why it matters, or what a valid value looks like. Need a 1-line explanation per field.
3. **Defaults invisible.** `MEGA_HOST` is hardcoded to `https://apibusiness1.megaapi.com.br` and never asked. `OPENAI_API_KEY` skip-on-empty is implicit. Should be shown as `[default: ...]` so users know what Enter does.
4. **megaAPI hardcoded in copy.** Comment says `--- megaAPI (https://megaapi.com.br) ---`. Other students use different WhatsApp HTTP providers (Evolution API, WPPConnect, Z-API, Baileys-based services). Wizard should ask for "WhatsApp HTTP API instance/token" generically. The host URL becomes a wizard field instead of a hardcoded value.
5. **No mention of `bd ready`.** Aligns with CLAUDE.md Rule 2 — track this work via beads before coding.

## Scope

In scope:
- `src/whatsapp_agent/setup_config.py` — every print + every `ask()` call.
- `MEGA_HOST` becomes a prompted field with a default shown.
- Output `config.py` template comments translated to English.
- Final "Proximos passos" block translated.

Out of scope (do NOT touch in this plan):
- Renaming `MEGA_*` symbols inside `config.py` / `webhook_server.py` / `send_message.py`. Those stay (`MEGA_HOST`, `MEGA_TOKEN`, etc.) — internal naming. User-facing wizard text is the only thing changing. Symbol rename is a separate refactor (would touch ~15 files + tests).
- `bootstrap.py` console output ("starting webhook_server", "tunnel up", etc.). Different file, separate pass.
- README / SETUP.md / CLAUDE_PROMPT.md.
- Other languages beyond English. No i18n framework — just rewrite strings.

## Non-goals

- Multi-language toggle (`--lang pt`). One default lang (English). PT speakers can still use it.
- Validating instance/token against an actual API endpoint. Future improvement (memory note already captured under "Wizard improvement").

## Design

### Field-by-field rewrite

Each field gets: **label**, **1-line explanation**, **default (shown in brackets)**, **example**.

| Order | Field | New prompt copy |
|-------|-------|-----------------|
| 1 | overwrite confirmation | `config.py already exists. Overwrite? [y/N]:` |
| 2 | header | `WhatsApp Claude Agent — Interactive Setup` |
| 3 | sub-header | `Fill in the values below. Press Enter to accept the default shown in [brackets].` |
| 4 | section: command token | `--- Optional command token (legacy) ---`<br>`Used to gate /commands. Leave blank — phone whitelist alone is enough for self-chat.` |
| 5 | `CMD_TOKEN` | `CMD_TOKEN [skip]:` |
| 6 | section: API host | `--- WhatsApp HTTP API ---`<br>`Base URL of your WhatsApp gateway provider (megaAPI, Evolution, WPPConnect, Z-API, etc.).` |
| 7 | `MEGA_HOST` | `API base URL [https://apibusiness1.megaapi.com.br]:` |
| 8 | `instance` | `Instance name (the ID/key your provider gave you):` |
| 9 | `token` | `Instance token (bearer token for the instance above):` |
| 10 | section: phone | `--- WhatsApp number ---`<br>`Your own WhatsApp number — the agent only replies to messages from/to this number.` |
| 11 | `phone` | `Phone in international format, digits only (e.g. 5511999999999):` |
| 12 | section: openai | `--- Audio transcription (optional) ---`<br>`Required only if you want to send voice messages. Get a key at https://platform.openai.com/api-keys.` |
| 13 | `OPENAI_API_KEY` | `OPENAI_API_KEY [skip]:` |
| 14 | success | `OK — config.py created.` |
| 15 | next steps | translated, English |

### Default-handling changes

`ask()` already supports `default=`. Two callsites need updating:
- `MEGA_HOST` — currently hardcoded. Add prompt with `default="https://apibusiness1.megaapi.com.br"`.
- `CMD_TOKEN` and `OPENAI_API_KEY` — already optional, just add visible `[skip]` hint in the label.

No `ask()` signature change needed.

### config.py template comment

Top comment block of generated `config.py`:

```python
# ============================================================
# WhatsApp Claude Agent — config.py (generated by setup_config.py)
# ============================================================
# DO NOT commit this file (already in .gitignore).
# To regenerate: delete config.py and run `python -m whatsapp_agent.setup_config`.
# ============================================================
```

`MEGA_*` variable names stay unchanged (out of scope).

### Next-steps block (English)

```
Next steps:
  1. Run: python scripts/bootstrap.py
     (starts webhook + Quick Tunnel + pushes URL to your provider — one command)
  2. Send a WhatsApp message to yourself
  3. Run: python -m whatsapp_agent.discover_lid
     (auto-detects your LID and updates config.py)
  4. Open Claude Code and paste in the first message:
     'Read CLAUDE_PROMPT.md and run the step 2 prompt. SESSION: 1'
```

Note step 4 references CLAUDE_PROMPT.md which is still PT. Aligning that file is out of scope here.

## Implementation steps

1. `bd create "Wizard UX + i18n: english copy, defaults shown, vendor-agnostic" -p 1` — track work.
2. `bd update <id> --claim`.
3. Edit `src/whatsapp_agent/setup_config.py`:
   - Replace every PT string with English version per table above.
   - Add `MEGA_HOST` prompt (was hardcoded).
   - Add `[skip]` hint on optional fields.
   - Update generated `config.py` header comment.
   - Update final "Next steps" block.
4. Update tests if any cover wizard output. (`grep -rn "Configuracao Interativa\|Preencha\|MEGA_HOST" tests/`).
5. Run `python -m pytest -q`. All green.
6. Manual smoke test: `cd /tmp && cp -r teste teste-wizard-test && cd teste-wizard-test && rm config.py && python -m whatsapp_agent.setup_config`. Walk through every field. Verify Enter accepts defaults. Verify generated config.py is valid Python.
7. `bd close <id> "<commit SHA>"`.
8. Commit on `trilha-1-multimodal`. Push. Sync `main`.

## Risks

- **Existing users with PT muscle memory.** Mitigation: copy stays semantically identical; field order unchanged; defaults preserved.
- **`MEGA_HOST` newly prompted.** Users hitting Enter get the current hardcoded value, so behavior unchanged for default users. Users with custom provider get the field they previously had to hand-edit.
- **Tests checking for PT strings.** Will fail loudly — easy to update.
- **CLAUDE_PROMPT.md still PT.** Documented as out-of-scope. Future PR.

## Verification

- Unit tests pass (`pytest -q`).
- Manual smoke test: wizard from blank state produces valid `config.py`.
- `python scripts/bootstrap.py` flows end-to-end on the smoke-tested `config.py`.
- Diff review: no symbol rename leaked into scope (only strings + one new prompt for `MEGA_HOST`).

## Rollback

Pure string-level refactor in one file (`setup_config.py`). Revert commit if anything breaks.
